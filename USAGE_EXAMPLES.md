# 使用範例

本文檔提供各種功能的實際使用範例。

## 功能範例

### 1. 簡體轉繁體

**輸入：**
```
这是一个简体中文的测试文本。我们需要将它转换成繁体中文。
计算机、网络、软件都是常见的词汇。
```

**輸出：**
```
這是一個簡體中文的測試文本。我們需要將它轉換成繁體中文。
計算機、網路、軟體都是常見的詞彙。
```

---

### 2. 英文加空白

**輸入：**
```
這是NextJS的範例，使用React19和TypeScript開發。
我們有2023個用戶，成長率達到50%。
```

**輸出：**
```
這是 NextJS 的範例，使用 React19 和 TypeScript 開發。
我們有 2023 個用戶，成長率達到 50%。
```

---

### 3. 修正錯字

**輸入：**
```
在在這個專案中，我們使用了佈局系統。
帐号設定需要在份內完成。
```

**輸出（根據字典）：**
```
再再這個專案中，我們使用了布局系統。
賬號設定需要在分內完成。
```

---

### 4. 刪除贅字

**輸入：**
```
然後，我想說其實這個功能基本上是很好的。
那麼，就是說我們呢需要測試一下。
我的的想法是這樣的的。
```

**輸出：**
```
我想說這個功能是很好的。
我們需要測試一下。
我的想法是這樣的。
```

---

### 5. 修正標點

**輸入：**
```
這是一個測試,,包含錯誤的標點..
請問你好嗎??今天天氣真好!!
"這是引號"，(這是括號)。
```

**輸出：**
```
這是一個測試，包含錯誤的標點。
請問你好嗎？今天天氣真好！
「這是引號」，（這是括號）。
```

---

### 6. 語義分段

**輸入：**
```
這是第一段的內容。這是第一段的第二句。這是第一段的第三句。這是第二段的內容。這是第二段的第二句。
```

**輸出（根據長度自動分段）：**
```
這是第一段的內容。這是第一段的第二句。這是第一段的第三句。

這是第二段的內容。這是第二段的第二句。
```

## 組合使用範例

### 情境：處理影片字幕稿

**輸入：**
```
[00:12] 然後,这个视频主要讲NextJS开发。
[00:34] 那么,我们来看看具体的代码!!
[01:23] 其实基本上这个框架很好用...
```

**啟用功能：**
- 簡體轉繁體
- 英文加空白
- 刪除贅字
- 修正標點
- 刪除時間戳

**輸出：**
```
這個影片主要講 NextJS 開發。
我們來看看具體的程式碼！
這個框架很好用⋯⋯
```

## 實用技巧

### 技巧 1: 批量處理
如果有多個文稿需要處理，可以：
1. 分別貼上處理
2. 使用「歷史記錄」功能快速查看之前的結果
3. 使用「匯出」功能下載所有處理後的文稿

### 技巧 2: 自訂字典
根據你的專業領域，自訂錯字字典：

```json
{
  "專業術語錯字": "正確術語",
  "常見打字錯誤": "正確拼寫"
}
```

### 技巧 3: 功能組合
不同情境建議的功能組合：

**處理簡體文章 → 繁體博客：**
- 簡體轉繁體
- 英文加空白
- 修正標點
- 語義分段

**處理語音轉文字稿：**
- 刪除贅字（移除口語填充詞）
- 修正標點
- 語義分段

**處理影片字幕：**
- 刪除時間戳
- 簡體轉繁體（如果是簡體字幕）
- 英文加空白
- 語義分段

## 常見問題

### Q1: 為什麼有些錯字沒有被修正？
A: 錯字修正功能依賴於字典檔案。如果某個錯字不在字典中，就不會被修正。您可以自行新增到 `/public/dictionaries/typo-dictionary.json`。

### Q2: 修正標點後，某些英文標點變成中文了？
A: 這是正常的。修正標點功能會根據上下文判斷。如果是純英文段落，建議先處理完其他功能後，手動調整標點。

### Q3: 語義分段把我的段落拆得太碎了？
A: 可以在 `lib/processors/segmentParagraphs.ts` 中調整 `maxParagraphLength` 參數，增加最大段落長度。

---

**提示：** 建議先用小段文字測試各功能，確認效果符合預期後，再處理完整文稿。
